# Quickstart

1. 前提
   - Go 1.22+ インストール済み
   - GitHub CLI (gh) が `gh auth login` 済み
   - プロジェクトIDを把握しておく

2. 取得と起動（想定フロー）
   - `go test ./...` で基盤ロジックの単体テストとゴールデン比較を実行
   - `go run ./cmd/projects-tui --project <id>` でTUIを起動（起動後にビュー切替/フィルタ/編集を確認）

3. 操作チェック
   - ビュー切替: `1/b`, `2/t`, `3/r`
   - ステータス移動: `h`/`l` (カンバン文脈)
   - フィルタ: `/` で条件入力、クリアで一覧復元
   - 編集/割当: `i` or `Enter` で編集、`a` で担当者

4. デバッグ/エラー時
   - gh出力エラーは非ブロッキングで表示される想定。再試行するかフィルタを緩和して確認。

5. 追加テストの目安
   - ロードマップ描画のゴールデン比較
   - gh呼び出しモックを用いたステータス更新/割当の統合テスト

6. 現状の補足
   - gh連携はスタブ実装のため、表示や状態更新はローカル反映で確認してください。
